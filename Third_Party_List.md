# Third Party List

This document lists the core tech stack, third-party software, and dependencies used in the `quip-rag` project. It includes specific version numbers, open-source licenses, and their usage scenarios/purposes within the project.

## 1. Backend Tech Stack (Python)

| Software / Dependency | Version | License | Purpose & Usage Scenario |
| :--- | :--- | :--- | :--- |
| **FastAPI** | `>=0.111.0` | MIT | **Core Backend Framework**: Provides a high-performance asynchronous RESTful API to handle frontend requests and orchestrate the RAG retrieval and chat workflow. |
| **Uvicorn [standard]** | `>=0.29.0` | BSD 3-Clause | **ASGI Server**: Contains `uvloop` (MIT) and `httptools` (MIT), serving as the runtime container for FastAPI to handle high-concurrency low-level network requests. |
| **Sentence-Transformers** | `>=3.0.0` | Apache 2.0 | **Embedding Model**: Used to chunk scraped Quip documents and convert them into text embeddings for high-dimensional semantic understanding. |
| **BeautifulSoup4** | `>=4.12.0` | MIT | **HTML Parser**: Parses and cleans rich-text HTML data fetched from the Quip platform, extracting pure text. |
| **lxml** | `>=5.0.0` | BSD | **Low-level Parsing Engine**: Works alongside BeautifulSoup to provide ultra-fast XML/HTML structural parsing capabilities. |
| **Pydantic** | `>=2.7.0` | MIT | **Data Validation**: Validates backend API request/response structures, enforces data types, and handles serialization. |
| **Pydantic-Settings** | `>=2.3.0` | MIT | **Configuration Management**: Reads `.env` variables and centrally manages sensitive information (e.g., model API Keys) and multi-environment configurations. |
| **python-multipart** | `>=0.0.9` | Apache 2.0 | **Form Data Support**: Extends FastAPI to receive and parse multipart form data or document-level uploads. |
| **aiosqlite** | `>=0.20.0` | MIT | **Lightweight Relational Storage**: Provides asynchronous local SQLite access for storing lightweight document metadata or chat history. |
| **httpx** | `>=0.27.0` | BSD 3-Clause | **Async HTTP Client**: Used to make asynchronous external network requests (e.g., calling LLM APIs, accessing external data sources). |
| **DuckDB** | `>=1.0.0` | MIT | **Embedded Analytical Database**: Executes high-performance structured data analysis and OLAP queries locally. |
| **LanceDB** | `>=0.6.13` | Apache 2.0 | **Embedded Vector Database**: Stores document vector matrices generated by the embedding model and performs efficient Approximate Nearest Neighbor (ANN) search during RAG QA. Replaces the traditional ChromaDB. |
| **Hatchling** | `(Build Dep)` | MIT | **Python Build Backend**: A modern PEP 517 compliant build system used to package and build project dependencies. |
